Лабораторная работа №9 — пункт 6: генерация EDM и сущностей (Database First / Model First)
============================================================================================

Общее: в Visual Studio для классического .NET Framework используется ADO.NET Entity Data Model
(файл с расширением .edmx). Пакет EntityFramework 6.x уже подключён к этому проекту.

--------------------------------------------------------------------------------------------
A) Database First (из существующей БД)
--------------------------------------------------------------------------------------------
1. В обозревателе решений: правый клик по проекту → Add → New Item.
2. Категория Data → ADO.NET Entity Data Model → имя, например ConfectioneryModel.edmx → Add.
3. Мастер: выберите «EF Designer from database» (модель из базы данных).
4. New Connection → укажите тот же сервер, что в App.config (LocalDB), база ConfectioneryLab.
5. Отметьте таблицы Category, Manufacturer, Product, Tag, ProductTag → Finish.
6. Visual Studio сгенерирует .edmx, сущностные классы (по умолчанию T4) и контекст, наследующий DbContext.

Для отчёта: приведите скриншот мастера и фрагмент сгенерированного контекста DbContext.

--------------------------------------------------------------------------------------------
B) Model First (сначала визуальная модель, потом БД)
--------------------------------------------------------------------------------------------
1. Add → New Item → ADO.NET Entity Data Model → имя, например ConfectioneryModelFirst.edmx → Add.
2. Выберите «Empty EF Designer model» (пустая модель).
3. Перетащите на диаграмму сущности Entity, задайте свойства и связи (Association) вручную.
4. Правый клик по области диаграммы → Generate Database from Model — получите SQL-скрипт создания БД.
5. Выполните скрипт на сервере (или через SQL Server Object Explorer), затем при необходимости
   обновите модель из БД (Update Model from Database).

Для отчёта: опишите шаги и приложите фрагмент сгенерированного SQL.

--------------------------------------------------------------------------------------------
Важно
--------------------------------------------------------------------------------------------
В текущем проекте для приложения используется Code First (папка Data_1\Ef) к той же базе,
чтобы не дублировать два контекста в одном exe. EDMX в отчёте можно хранить в отдельной
ветке или в копии проекта «демо-EDMX», чтобы не смешивать две технологии в одной сборке.
