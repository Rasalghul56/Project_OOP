Подготовка перед защитой
-------------------------
• Собрать проект (Release/Debug), убедиться что в bin\Debug рядом с exe лежат
  EntityFramework.dll и EntityFramework.SqlServer.dll (копируются из packages).
• Установлен SQL Server Express LocalDB; при первом запуске БД создаётся сама.
• Знать пароль/роль: в главном окне в комбобоксе «Роль» выбрать АДМИНИСТРАТОР
  (иначе пункт меню базы не откроется).


--------------------------------------------------------------------------------
ЧАСТЬ A — ЖИВОЙ ПОКАЗ В ПРИЛОЖЕНИИ (порядок нажатий и что говорить)
--------------------------------------------------------------------------------

1) Запуск и вход в зону лаб. 9
   • Запустить Lab_06_OOP.exe.
   • Фраза для защиты: «Основной каталог товаров по-прежнему работает через ADO.NET
     из лабораторной 8; блок Entity Framework вынесен в админ-панель, чтобы не
     дублировать два доступа к данным в одном сценарии пользователя.»
   • Внизу/сбоку панели: ComboBox «Роль» → выбрать «Администратор» (или как подписано в ru/en).
   • Меню → пункт про базу данных (заголовок из ресурсов, обычно про таблицы БД).
   • Откроется окно «Таблицы БД (лаб. 8–9)». Пролистать вниз до жёлтой рамки
     «Entity Framework — лаб. 9».

2) П.1–2 ТЗ (NuGet + конфиг) — устно + файл на экране
   • Не нажимая кнопок: указать мышкой на подсказку в окне про App.config.
   • Открыть в VS рядом: App.config — показать connectionStrings name="Confectionery",
     appSettings, configSections + entityFramework.
   • Комментарий: «Строка подключения и параметры в конфигурации; секция entityFramework
     зарегистрирована в configSections, иначе CLR не примет конфиг при старте.»

3) П.3–4 ТЗ (Code First + CRUD) — код, потом одна кнопка
   • В VS открыть Data_1\Ef\ConfectioneryDbContext.cs — связи 1-ко-многим и
     HasMany–WithMany для Product–Tag.
   • Открыть Data_1\Ef\ConfectioneryEfService.cs — коротко показать AddProduct,
     UpdateProduct, DeleteProduct, GetProductById.
   • В окне приложения нажать: «EF: CRUD метка».
   • В поле результата появится текст про создание/редактирование/удаление тестовой метки.
   • Комментарий: «CRUD показан на сущности Tag и связи многие-ко-многим с товаром,
     чтобы не портить демо-данные основного каталога.»

4) П.5 ТЗ — LINQ, фильтр, несколько полей, async, транзакция (по шагам)
   Шаг A — async + просмотр
   • Кнопка: «EF: загрузить товары (async)».
   • Комментарий: «ToListAsync, Include навигаций, AsNoTracking для чтения.»

   Шаг B — фильтрация + поиск по нескольким словам
   • В поле «Мин. цена:» ввести: 2
   • В «Часть названия категории:» ввести: Пиро   (или Торт — по данным в БД)
   • В «Слова поиска (несколько через пробел):» ввести: крем шоколад
     (подберите слова, которые реально есть в ShortName/FullName/Description сидов)
   • Кнопка: «EF: LINQ поиск».
   • Комментарий: «Один запрос IQueryable: цена от порога, LIKE по категории,
     каждое слово дополнительно фильтрует по полям товара — это LINQ to Entities.»

   Шаг C — сортировка
   • Кнопка: «EF: сортировка по цене».
   • Комментарий: «OrderBy по цене в сервисе, результат в строке ниже кнопок.»

   Шаг D — async подсчёт
   • Кнопка: «EF: async на складе».
   • Комментарий: «CountAsync по условию категория и Quantity>0 — асинхронный LINQ.»

   Шаг E — транзакция
   • Кнопка: «EF: транзакция (ROLLBACK)».
   • Комментарий: «BeginTransaction, INSERT категории с уникальным именем, SaveChanges,
     Rollback — в БД запись не остаётся; проверка вторым контекстом в коде.»

5) П.6 ТЗ — Database First / Model First (без живого EDMX в этом exe)
   • Открыть файл Data_1\Лаб9_EDM_генерация.txt на экране.
   • Комментарий: «В рабочем проекте используется Code First к той же БД. Генерация
     EDMX из базы и из пустой модели описана пошагово для отчёта; при необходимости
     модель .edmx делается копией проекта в Visual Studio по инструкции.»

ЧАСТЬ B — КАКИЕ ФАЙЛЫ ПОКАЗЫВАТЬ И КАКОЙ ФРАГМЕНТ КОДА ИМЕЕТ СМЫСЛ

--------------------------------------------------------------------------------
ЧАСТЬ C — КРАТКИЕ ФРАЗЫ «ЧТО ОТВЕТИТЬ ПРЕПОДАВАТЕЛЮ» ПО ПУНКТАМ ТЗ
--------------------------------------------------------------------------------

П.1  «Entity Framework 6.4 подключён через NuGet: packages.config и ссылки в csproj
      на EntityFramework и EntityFramework.SqlServer.»

П.2  «Параметры в App.config: appSettings, connectionStrings; для EF добавлена
      секция entityFramework с провайдером SqlClient.»

П.3  «Code First: классы сущностей в Data_1\Ef, маппинг на таблицы dbo.*; связи
      один-ко-многим Category–Product и Manufacturer–Product; многие-ко-многим
      Product–Tag через таблицу ProductTag, настроено в OnModelCreating.»

П.4  «CRUD для товара — методы в ConfectioneryEfService; на защите показываю CRUD
      по метке с демо-кнопкой, чтобы не удалять реальные товары.»

П.5  «Сортировка и фильтрация — LINQ to Entities в том же сервисе; поиск по одному
      полю реализован в SearchOneField, по нескольким — FilterAndSearchMulti с
      разбиением строки на слова; асинхронность — ToListAsync и CountAsync;
      транзакция — Database.BeginTransaction с откатом в демо-методе.»

П.6  «Генерация EDMX Database First и Model First описана в Лаб9_EDM_генерация.txt,
      в основном проекте не подключал второй контекст EDMX, чтобы не конфликтовать
      с Code First в одной сборке.»
